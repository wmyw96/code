然而，我们不能通过之前的对于最大流的一个认识（即一条流对应一种方案）来解决这个问题，这是不对的。

那么，这怎么做呢？

如果撇开最大流的因素，我们考虑如果存在欧拉回路必然对于每个点来说它的入度=出度，这是充分必要条件。

如果有向边决定的入度出度已经决定了，我们只能改变无向边的入度出度。

但是，这样建图又进入了死胡同。

我们发现网络流的算法就是一个不断更新的过程，那么，我们是不是可以初始化（也就是随机）定无向边的方向呢？

继续考虑，对于已经建好的图，如果一个点的入度-出度是个奇数的话，显然不合法（因为每次改变会使其差距-2)。

那么，接下来，我们该怎么办？

连(i,T,(in-out)/2)，当且仅当in-out>0 (需要出去)

连(S,i,(out-in)/2)，当且仅当out-in>0（需要进来）

那么，我们继续想该如何改变。

我们发现，只有无向边才能改变ret(i)=(in-out)/2,那么，对于一开始，如果我设成了<x,y>，如果我要改的话，那么必然是把x的ret+1，把y的ret-1――好吧，从中可以显而易见地得出，这个边是要有方向的（而且是<x,y>这让x的一个单位的ret给y）。
