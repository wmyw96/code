最小割分析（2）

显然这是一个经典的问题，求最大闭合子图。

对于这类问题，黑书上有一个非常好的方法，大概是这样的。

我们对于正的点i连（S,i），容量为他的权值w，对于负的点连（i,T），容量为他的权值的相反数-w，然后，对于原图中的边保留，容量为SUP(这是上确界的意思)

然后，我们在求了最小割之后，我们考虑最小割的意义：

对于S集里面的每一个非s的点，实际意义上我们对于这些点都是要取的，所以，最小割的**实际意义就是对于[没取的点的正权值]和[取了的点负权值的相反数]之和**，然后（所有的正权值之和-最小割）就是答案。

第二问，似乎很烦，求最小的点数，显然，通过yy我们可以知道，闭合子图不需要互相连通，所以，唯一的可能是点数增加的方法就是取了权值和为0的子图，然后，在网络流里面显然就会是满流，可以不予考虑。这样的话，就只要求S集里的点之和-1即可（实际操作就是从S遍历残余网络，能遍历到的就遍历。）
